# 횡스크롤 슈팅 게임 상세 사양서

## 1. 게임 개요

### 1.1 기본 정보
- **게임명**: Sky Defender (스카이 디펜더)
- **장르**: 횡스크롤 슈팅 게임 (Horizontal Shoot 'em up)
- **플랫폼**: Web (HTML5 Canvas, 데스크톱 브라우저)
- **대상 플레이어**: 캐주얼 게이머, 8세 이상
- **게임 언어**: 한국어 (또는 다국어 지원 예정)

### 1.2 게임의 핵심 개념
플레이어는 우주 전투기 조종사가 되어 적 우주선들의 공격으로부터 지구를 방어합니다.
- **핵심 루프**: 적 피하기 → 적 격추 → 점수 획득 → 난이도 상승
- **주요 특징**:
  - 간단하지만 중독성 있는 게임플레이
  - 시간이 지날수록 어려워지는 난이도 진행
  - 하이스코어 시스템으로 재플레이 유도

### 2.2 플레이어 액션 목록

| 액션 | 설명 | 입력 |
|------|------|------|
| **좌측 이동** | 플레이어 기체를 좌측으로 이동 | A 키 또는 ← 화살표 |
| **우측 이동** | 플레이어 기체를 우측으로 이동 | D 키 또는 → 화살표 |
| **발사** | 플레이어의 총알 발사 | SpaceBar 또는 클릭 |
| **일시 정지** | 게임 일시 정지 (v1.0에서는 미지원) | P 키 |
| **음량 조절** | 게임 음량 조절 (v1.0에서는 미지원) | M 키 |

### 2.3 게임 진행 방식

1. **게임 시작**: 플레이어는 화면 하단 중앙에 생성됨
2. **적 생성**: 화면 상단에서 일정한 간격으로 적이 생성됨
3. **이동 및 발사**: 플레이어가 키보드로 움직이면서 발사
4. **점수 증가**: 적을 격추할 때마다 점수 증가
5. **체력 감소**: 적 총알에 맞거나 적과 충돌하면 체력 감소
6. **난이도 상승**: 시간이 지남에 따라 (또는 점수에 따라) 적의 속도, 생성 빈도 증가
7. **게임 종료**: 체력이 0이 되면 게임 오버

## 4. 적 시스템

### 4.1 적의 종류

#### 4.1.1 기본 적 (Basic Enemy)
- **외형**: 작은 우주선 모양 (적색)
- **크기**: 25 × 25 px
- **체력**: 1 HP
- **속도**: 시간에 따라 증가 (초기 100 px/s)
- **행동 패턴**: 직선 하강 (위에서 아래로)

#### 4.1.2 강화 적 (Strong Enemy) - Wave 5부터 등장
- **외형**: 더 큰 우주선 (주황색)
- **크기**: 35 × 35 px
- **체력**: 3 HP
- **속도**: 기본 적의 80% 속도
- **행동 패턴**: 직선 하강 + 약간의 좌우 진동
- **발사**: 플레이어를 향해 2개의 총알을 발사

### 4.2 적 생성 규칙

| Wave | 시간 | 적 생성 간격 | 적 종류 | 난이도 배수 |
|------|------|------------|--------|-----------|
| 1-2 | 0-20s | 1.2초 | 기본 적만 | 1.0x |
| 3-4 | 20-40s | 1.0초 | 기본 적만 | 1.2x |
| 5-6 | 40-60s | 0.8초 | 기본 적 80%, 강화 적 20% | 1.4x |
| 7-10 | 60-90s | 0.6초 | 기본 적 60%, 강화 적 40% | 1.6x |
| 11+ | 90s이상 | 0.5초 | 기본 적 40%, 강화 적 60% | 1.8x |

**Wave 결정 방식**: 누적 점수 / 1000 = Wave 번호 (최대 Wave 15)

### 4.3 적의 이동 및 행동

```javascript
// 기본 적 이동
적_Y좌표 += 적_속도 * deltaTime
적_X좌표 = 초기_X좌표 (고정)

// 화면을 벗어나면 제거
if (적_Y좌표 > canvas.height) {
  적 제거
}
```

#### 강화 적 진동 패턴 (선택사항)
```javascript
적_X좌표 = 초기_X좌표 + Math.sin(경과시간 * 2) * 30
```

### 4.4 적의 발사 패턴 (강화 적만)

| 항목 | 값 |
|------|-----|
| **발사 조건** | 플레이어와 Y좌표가 가까워질 때 |
| **발사 간격** | 1.5초 마다 |
| **발사 개수** | 2개 (30도 좌우 분산) |
| **총알 속도** | 150 px/s |
| **총알 크기** | 8 × 8 px (적색) |

## 6. 충돌 및 데미지 시스템

### 6.1 충돌 감지 방식

**사각형 기반 충돌 감지 (AABB - Axis-Aligned Bounding Box)**

```javascript
function 충돌감지(A, B) {
  return A.x < B.x + B.width &&
         A.x + A.width > B.x &&
         A.y < B.y + B.height &&
         A.y + A.height > B.y;
}
```

### 6.2 충돌 조건 및 결과

#### 6.2.1 플레이어 총알 + 기본 적

| 충돌 전 | 충돌 후 |
|---------|--------|
| 플레이어 총알 1개 | 제거됨 |
| 기본 적 1개 (1 HP) | 제거됨 |
| 점수 | +10점 |
| 효과음 | 폭발음 (explosion) |
| 파티클 | 황색 폭발 (3-5개 파티클) |

#### 6.2.2 플레이어 총알 + 강화 적

| 충돌 전 | 충돌 후 |
|---------|--------|
| 플레이어 총알 1개 | 제거됨 |
| 강화 적 (N HP) | 체력 -1 |
| 점수 | +5점 (체력당 1회) |
| 효과음 | 타격음 (hit) |
| 파티클 | 주황색 스파크 (1-2개) |

**강화 적 완전 격추 시**: 추가 +20점

#### 6.2.3 적 총알 + 플레이어

| 충돌 전 | 충돌 후 |
|---------|--------|
| 적 총알 | 제거됨 |
| 플레이어 체력 | -1 |
| 점수 | 변화 없음 |
| 효과음 | 피격음 (damage) |
| 파티클 | 적색 폭발 |
| 무적 시간 | 2초 (깜빡이는 표시) |

#### 6.2.4 적 + 플레이어 (접촉)

| 충돌 전 | 충돌 후 |
|---------|--------|
| 적 | 제거됨 |
| 플레이어 체력 | -1 |
| 점수 | 변화 없음 |
| 효과음 | 폭발음 (crash) |
| 파티클 | 큰 폭발 (5-8개) |
| 무적 시간 | 2초 |

#### 6.2.5 화면 범위 이탈

| 객체 | 결과 |
|------|------|
| 플레이어 | 경계 내에서만 이동 가능 (이탈 불가) |
| 적 | 화면 하단 벗어나면 자동 제거 |
| 플레이어 총알 | 화면 상단 벗어나면 자동 제거 |
| 적 총알 | 화면 하단 벗어나면 자동 제거 |

### 6.3 점수 계산 공식

```
기본 점수 = (기본 적격추 * 10) + (강화 적격추 * 20) + (강화 적 타격 * 5)

보너스 점수 = floor(현재_Wave * 2)

최종 점수 = 기본 점수 + 보너스 점수
```

## 7. 점수 및 게임 오버 시스템

### 7.1 점수 획득 방식

| 이벤트 | 점수 | 조건 |
|--------|------|------|
| **기본 적 격추** | +10 | 플레이어 총알이 기본 적과 충돌 |
| **강화 적 격추** | +20 | 플레이어 총알이 강화 적의 마지막 체력을 제거 |
| **강화 적 타격** | +5 | 플레이어 총알이 강화 적의 체력을 1 감소 |
| **Wave 보너스** | +2×Wave | 각 Wave마다 자동 추가 (누적) |

### 7.2 게임 오버 조건

| 조건 | 결과 |
|------|------|
| **플레이어 체력 <= 0** | 게임 즉시 종료 |
| **플레이어 사망 후 3초** | 게임 오버 화면 전환 |

### 7.3 게임 오버 화면 구성

최종 점수, 하이스코어, 다시 시작 버튼 표시

### 7.4 게임 리셋 방식

**[다시 시작] 클릭 시**:
1. 모든 적 제거
2. 모든 총알 제거
3. 플레이어 위치 초기화 (중앙 하단)
4. 플레이어 체력 3으로 초기화
5. 점수 0으로 초기화
6. 경과 시간 0으로 초기화
7. Wave 1로 초기화
8. 게임플레이 화면으로 전환
